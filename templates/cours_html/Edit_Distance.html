Introduction and Motivations

The concept of Edit Distance, also known as Levenshtein Distance, emerged in the late 1960s through the works of Vladimir Levenshtein. This metric has since become indispensable in various fields such as bioinformatics, natural language processing, and computer science. The edit distance between two strings is a measure of the dissimilarity between them, defined as the minimum number of single-character edits (insertions, deletions, or substitutions) required to change one string into the other.

The notion of edit distance is crucial in scenarios where we need to quantify how different two strings are. For instance, in spell-checking algorithms, edit distance helps in identifying the closest match for a misspelled word. In bioinformatics, it aids in comparing DNA sequences and identifying mutations.

Definitions

To understand edit distance, let’s first consider two strings \(s\) and \(t\). We aim to find the minimum number of operations required to transform \(s\) into \(t\). The allowed operations are:

  • Insertion: Adding a character to \(s\).

  • Deletion: Removing a character from \(s\).

  • Substitution: Replacing a character in \(s\) with another character.

Formally, the edit distance \(d(s, t)\) between two strings \(s\) and \(t\) is defined as:

\[d(s, t) = \min \{ |s| + |t| - |lcs(s, t)| \}\]

where \(lcs(s, t)\) denotes the longest common subsequence between \(s\) and \(t\).

Alternatively, we can define edit distance using dynamic programming. Let \(s = s_1s_2 \dots s_m\) and \(t = t_1t_2 \dots t_n\). The edit distance can be computed using the following recurrence relation:

\[d(s, t) = \begin{cases} m & \text{if } n = 0, \\ n & \text{if } m = 0, \\ d(s_{1..m-1}, t_{1..n-1}) & \text{if } s_m = t_n, \\ 1 + \min \{ d(s_{1..m-1}, t), d(s, t_{1..n-1}), d(s_{1..m-1}, t_{1..n-1}) \} & \text{otherwise.} \end{cases}\]

Theorems

One of the fundamental theorems related to edit distance is the Levenshtein Distance Theorem, which states that the edit distance between two strings can be computed efficiently using dynamic programming.

For any two strings \(s\) and \(t\), the edit distance \(d(s, t)\) can be computed in \(O(mn)\) time and space complexity, where \(m\) and \(n\) are the lengths of \(s\) and \(t\), respectively.

Proof. We can construct a matrix \(D\) of size \((m+1) \times (n+1)\), where each entry \(D[i][j]\) represents the edit distance between the first \(i\) characters of \(s\) and the first \(j\) characters of \(t\). The base cases are:

\[D[i][0] = i \quad \text{for } 0 \leq i \leq m,\] \[D[0][j] = j \quad \text{for } 0 \leq j \leq n.\]

For \(1 \leq i \leq m\) and \(1 \leq j \leq n\), we have:

\[D[i][j] = \begin{cases} D[i-1][j-1] & \text{if } s_i = t_j, \\ 1 + \min \{ D[i-1][j], D[i][j-1], D[i-1][j-1] \} & \text{otherwise.} \end{cases}\]

The value \(D[m][n]\) gives the edit distance between \(s\) and \(t\). The time complexity is \(O(mn)\) since we fill the matrix in a bottom-up manner. The space complexity is also \(O(mn)\) due to the storage requirements of the matrix. ◻

Properties and Corollaires

  1. The edit distance satisfies the triangle inequality. For any three strings \(s\), \(t\), and \(u\), we have:

    \[d(s, u) \leq d(s, t) + d(t, u).\]

  2. The edit distance is a metric. It satisfies the following properties:

    • Non-negativity: \(d(s, t) \geq 0\).

    • Identity of indiscernibles: \(d(s, t) = 0\) if and only if \(s = t\).

    • Symmetry: \(d(s, t) = d(t, s)\).

The edit distance between two strings can be used to measure the similarity between them. A lower edit distance indicates higher similarity.

Proof. The proof follows directly from the definition of edit distance. If two strings have a small edit distance, it means that few operations are needed to transform one into the other, indicating that they are similar. ◻

Conclusion

The edit distance is a powerful and versatile metric with applications in various fields. Its efficiency and simplicity make it an essential tool for comparing strings and quantifying their dissimilarity. The dynamic programming approach provides an optimal way to compute the edit distance, ensuring its practical utility in real-world applications.
